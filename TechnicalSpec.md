## 一、技术选型

### 1.1 约束条件

| 约束 | 说明 |
|------|------|
| **使用场景** | 本地个人使用 |
| **成本要求** | 0资金成本 |
| **无需部署** | 直接浏览器打开使用 |
| **无需服务器** | 纯静态前端 |

### 1.2 技术栈选择

#### 最终方案：HTML + Vanilla JavaScript（零依赖）

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **页面结构** | 纯HTML | 轻量、无需构建 |
| **样式** | Vanilla CSS | 无需预处理器 |
| **逻辑** | Vanilla JavaScript | 无需框架，纯原生 |
| **数据存储** | LocalStorage | 浏览器本地存储 |
| **部署方式** | 双击HTML直接打开 | 零成本运行 |

#### 备选方案（后期如需更复杂功能）

| 场景 | 技术选型 |
|------|----------|
| 需要组件化 | Vue 3（CDN引入） |
| 需要状态管理 | Pinia |
| 需要路由 | Vue Router |
| 需要构建工具 | Vite |

### 1.3 无成本方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **HTML单文件** | 双击即用、零依赖 | 代码量大会乱 | MVP阶段 |
| **Vue CDN** | 组件化开发 | 需网络加载 | 中期版本 |
| **Vite构建** | 开发体验好 | 需要Node.js | 后期上线 |

**结论**：MVP阶段采用纯HTML单文件方案，后期考虑升级。

## 七、性能优化

### 7.1 加载优化

| 优化点 | 方案 |
|------|------|
| 首屏渲染 | 内联关键CSS |
| 题库加载 | 按需加载 |
| 图片资源 | 使用CSS绘制，无图片 |

### 7.2 运行优化

| 优化点 | 方案 |
|------|------|
| DOM操作 | 减少重排，使用DocumentFragment |
| 事件处理 | 事件委托 |
| 存储 | 批量写入，减少IO |

### 7.3 预期性能

| 指标 | 预期值 |
|------|--------|
| 文件大小 | < 200KB（不含题库） |
| 首次加载 | < 1秒 |
| 答题响应 | < 50ms |
